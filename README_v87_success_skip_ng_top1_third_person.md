# v87 success-skip-ng-top1 + third-person hand toggle

## 修正1: 成立時はNG経路を処理しない

方針:
- API一括判定後、成立候補が1件以上あれば成功処理だけ行う。
- NG経路は表示しない。
- NG理由jobを作らない。
- NG pollingもしない。
- 成功表示・ログ・読み上げ・画像だけに集中する。

狙い:
- 「成立したのに別候補NGも表示される」混乱を排除。
- reason job 増殖、解析待ち残り、無駄なAPI処理を削減。

## 修正2: 成立0のときだけ代表NG TOP1を処理

方針:
- 成立候補が0件のときだけ、代表NG経路TOP1を選ぶ。
- 2件目以降のNG経路は表示・理由解析・pollingしない。
- TOP1は長い経路優先、同点なら盤面上の先頭位置優先。

狙い:
- NG候補が大量に画面を埋める問題を止める。
- 理由解析を1本だけに限定して、待ち/解析中の増殖を防ぐ。

## 修正3: 選択手札の三人称単数現在形 化/戻す ボタン

追加UI:
- `3単現 化/戻す` ボタンを追加。

挙動:
- 手札を選択して押すと、動詞を三単現へ変化する。
  - like -> likes
  - play -> plays
  - study -> studies
  - go -> goes
  - have -> has
  - am/are -> is
- もう一度押すと元の手札へ戻す。
- 変化中の手札は選択表示に `（3単現）` を付ける。
- 判定は従来どおり外部APIのみ。三単現ボタンは手札操作UIであり、文法判定ハードコードではない。
