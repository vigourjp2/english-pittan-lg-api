# v25 理由解析キュー詰まり対策

## 修正内容

- 理由解析ワーカーは1本固定のまま。
- `REASON_JOB_MAX_ATTEMPTS` を1〜6回に強制クランプ。
  - Render環境変数に `999999` 等が残っていても無限リトライしない。
- processReasonQueue の先頭で、既に最大試行回数を超えた古いjobを `failure` に落とす。
  - `再試行1469` のような旧jobが残っても、次回処理で即終了扱いにする。
- 次job選択時に `failure/error/cancelled` を除外。
- `/health` に `maxAttempts` と `rawMaxAttempts` を表示。
  - 実際の上限値と、環境変数の生値を確認できる。
- ブラウザ側の Link Grammar キャッシュキーを v25 に更新。
  - 古い `再試行1469` 表示キャッシュを切る。

## 期待動作

- 最新の1件だけ `理由解析中…`。
- 次候補は `理由解析待ち①`、次は `理由解析待ち②`。
- 古いjobは、現在実行中の1件が終わったタイミングで再ソートされ、新しい一手より後ろに回る。
- 最大試行回数を超えたjobは `理由解析失敗` になり、キュー先頭に戻らない。

## 確認

- Render再デプロイ後、`/health` の `mode` が `v25` になっていること。
- `/health` の `reasonJobs.maxAttempts` が 1〜6 の範囲になっていること。
- game URL は `?v=25` を付けて開く。
