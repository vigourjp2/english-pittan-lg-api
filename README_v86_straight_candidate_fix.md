# v86 straight candidate extraction fix

## 原因
v74/v85 の `scanFromCell()` は、盤面上の全カードを開始点にして「右または下へ進める限りDFSでつなぐ」実装になっていた。
そのため、プレイヤーが直線に置いたつもりでも、別行・別列の隣接カードまで L字/階段状に勝手に接続され、以下のような候補が大量生成された。

- `I am happy she speak apples`
- `I am happy she see apples`
- `I am kind they see apples`

これは API 判定の問題ではなく、フロント側の候補抽出が広すぎる問題。
APIは渡された候補を判定していただけ。

## 修正
- `scanFromCell()` から右/下DFSによる任意ルート候補生成を撤去。
- 英文候補は、今回置いたカードを含む「横一直線」または「縦一直線」の連続カードだけに限定。
- 判定は引き続きAPIのみ。ローカル文法ハードコードは追加しない。
- 上部メッセージのNG候補表示は代表3件までに制限し、残りは `他N件` と表示。

## 期待挙動
- 別行/別列をまたいで勝手に長文候補を作らない。
- 置いたカードを含む横列・縦列だけを一括API判定する。
- 不成立時の表示が長文NG候補で埋まらない。
