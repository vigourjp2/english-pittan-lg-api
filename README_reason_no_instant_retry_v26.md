# v26 理由解析：一瞬で再試行4になる不具合修正

## 原因
v25 では、`I am` など同じ英文の理由解析 job が一度 `failure(attempts=4)` になると、同じキーの job がサーバーのメモリに残っていた。
その後、新しいゲーム・新しい一手で同じ `I am` が発生しても、既存の失敗済み job を再利用していたため、 attempts が 0 に戻らず、一瞬で「理由解析失敗（再試行4）」表示になっていた。

また、retry delay の先頭に `0ms` が入っていたため、失敗条件によっては初回再試行が即走り、ユーザーから見ると一瞬で進んだように見えた。

## 修正
- 失敗済み terminal job を新しい一手で再要求した場合、attempts=0 の新 job として作り直す。
- retry delay の先頭 0ms を廃止し、標準を `3000,8000,15000,30000` に変更。
- 標準最大試行回数を 3 回に変更し、環境変数でも最大 5 回までに clamp。
- running 中 job は途中キャンセルしないが、完了直後に新しい一手の job を優先して再ソートする方針は維持。

## 確認
- `/health` の mode が `link-grammar-reason-job-v26-no-instant-terminal-reuse` であること。
- ゲーム URL は `?v=26` を付けて開くこと。
